Learn MIX step by step

01Hello World

Every MIX program starts with a main() function. There is no fn keyword -- just write the function name, parentheses for parameters, and indent the body with 4 spaces. MIX uses indentation to define blocks, similar to Python.

Use the built-in print() function to write text to the console. String literals are enclosed in double quotes. There are no semicolons in MIX.

main()
    print("Hello, world!")mix

Save this as hello.mix and compile it:

# Compile and run in one step
mix run hello.mix

# Or compile only, then run separately
mix build hello.mix
./helloshell

The mix run command compiles your program and immediately executes it. The mix build command compiles to a native binary without running it. The output binary name is derived from the source filename by default, or you can specify one with -o.

02Variables and Types

Variables in MIX are immutable by default. Use = for immutable bindings and ! after the name for mutable variables. This makes it immediately visible which values can change, improving readability and preventing accidental mutation.

MIX has four primitive types: int, float, str, and bool. Types are inferred from the assigned value, but you can add an explicit type annotation with a colon when you want to be specific.

main()
    // Immutable variables (default)
    name = "Alice"
    age = 30
    pi = 3.14159
    active = true

    // Mutable variables (note the ! after the name)
    count! = 0
    count! = count + 1

    // Explicit type annotation
    score: float = 99.5
    label: str = "player"

    // Compound assignment on mutable variables
    total! = 10
    total! += 5      // total is now 15
    total! -= 3      // total is now 12

    print("{name} is {age} years old")
    print("count = {count}, total = {total}")mix

The ! suffix is a core MIX convention: it marks anything that mutates state. You will see it on variables, method names like push!(), and parameters that get modified.

03Functions

Functions are defined by writing the name, parameters in parentheses, and an optional return type after ->. The body is indented. There is no fn or def keyword. The last expression in the body is implicitly returned.

Use done for early return from a function. Functions that perform side effects (I/O, mutation, printing) should be marked with ~ after the closing parenthesis to signal to callers that the function has effects beyond its return value.

// Simple function with return type
square(x: int) -> int
    x * x

// Multiple parameters
add(a: float, b: float) -> float
    a + b

// Early return with done
abs_val(n: int) -> int
    if n >= 0
        done n
    -n

// Side-effect marker ~ (this function prints)
greet(name: str) ~
    print("Hello, {name}!")

main()
    result = square(5)
    print("5 squared = {result}")

    sum = add(2.5, 3.7)
    print("sum = {sum}")

    greet("MIX")mix

Functions without a -> return type implicitly return nothing (void). The implicit return of the last expression means you rarely need done -- it is mainly useful for early exits in conditional logic.

04Control Flow

MIX provides if/else for conditionals, while for condition-based loops, for with ranges for counted loops, and for-each for iterating over collections. All blocks use indentation -- there are no braces or end keywords.

Ranges use .. for exclusive upper bound and ..= for inclusive. The match expression handles pattern matching with _ as the wildcard catch-all.

main()
    // if / else
    x = 42
    if x > 100
        print("big")
    else if x > 10
        print("medium")
    else
        print("small")

    // while loop
    count! = 0
    while count < 5
        print("count = {count}")
        count! += 1

    // for with exclusive range (0, 1, 2, 3, 4)
    for i in 0..5
        print("i = {i}")

    // for with inclusive range (1, 2, 3, 4, 5)
    for n in 1..=5
        print("n = {n}")

    // for-each over a list
    colors = ["red", "green", "blue"]
    for c in colors
        print("color: {c}")

    // match with wildcard
    day = "Monday"
    match day
        "Saturday", "Sunday" => print("weekend")
        _ => print("weekday")mix

You can also use break and continue inside loops just like in C or Python. The repeat keyword creates an infinite loop that runs until a break.

05Strings

Strings in MIX are enclosed in double quotes. They support interpolation with {} braces -- any expression inside braces is evaluated and converted to its string representation. Concatenation uses the + operator.

MIX strings come with a rich set of built-in methods. Methods that mutate the string in place are suffixed with !. Use to_string() to convert other types to their string representation.

main()
    // Literals and interpolation
    name = "MIX"
    greeting = "Hello, {name}!"
    print(greeting)  // Hello, MIX!

    // Concatenation
    full = "Hello" + " " + "World"

    // String methods
    s = "  Hello, World!  "
    print(s.len)                // 17
    print(s.trim)               // "Hello, World!"
    print(s.upper)              // "  HELLO, WORLD!  "
    print(s.lower)              // "  hello, world!  "

    msg = "hello world"
    print(msg.contains("world"))    // true
    print(msg.starts_with("hello")) // true
    print(msg.ends_with("world"))   // true
    print(msg.index_of("world"))    // 6

    // split, replace, slice
    csv = "a,b,c,d"
    parts = csv.split(",")       // ["a", "b", "c", "d"]

    fixed = msg.replace("world", "MIX")
    print(fixed)                    // "hello MIX"

    print(msg.char_at(0))           // "h"
    print(msg.slice(0, 5))          // "hello"
    print(msg.repeat(2))            // "hello worldhello world"
    print(msg.reverse)              // "dlrow olleh"
    print("dcba".sort())            // "abcd"

    // Convert other types to string
    n = 42
    s2 = to_string(n)              // "42"

    // String comparison (lexicographic)
    print("apple" < "banana")    // true
    print("hello" == "hello")    // true

    // Character / Unicode operations
    print(ord("A"))               // 65
    print(chr(65))                // "A"
    print("Z".code())            // 90mix

06Lists

Lists are ordered, dynamically-sized collections written with square brackets. They are typed -- all elements must be the same type. Access elements by index (zero-based) and use .len to get the count.

Mutation methods end with ! to make it clear they modify the list in place. List comprehensions let you build new lists with a concise for/if syntax inside brackets.

main()
    // Create a list
    nums! = [10, 20, 30, 40, 50]
    print(nums.len)        // 5
    print(nums[0])         // 10
    print(nums[4])         // 50

    // Mutating methods (note the !)
    nums.push!(60)         // [10, 20, 30, 40, 50, 60]
    last = nums.pop!()     // last = 60, list shrinks
    nums.insert!(0, 5)     // [5, 10, 20, 30, 40, 50]
    nums.remove!(0)        // [10, 20, 30, 40, 50]
    nums.sort!()            // sorts int, float, and str lists
    nums.reverse!()         // reverse in place

    // Non-mutating methods
    print(nums.contains(30))  // true
    print(nums.index_of(30))  // 2

    // Join list of strings
    words = ["hello", "world"]
    print(words.join(", "))   // "hello, world"

    // Slicing (creates a new list)
    mid = nums[1..3]       // elements at index 1, 2

    // List comprehension
    squares = [x * x for x in 1..6]
    print(squares)           // [1, 4, 9, 16, 25]

    // Filtered comprehension
    evens = [x for x in 1..11 if x % 2 == 0]
    print(evens)             // [2, 4, 6, 8, 10]

    // Iterate
    for n in nums
        print("item: {n}")mix

07Maps

Maps are key-value collections written with curly braces. Keys are strings and values can be any consistent type. Access values by key using bracket notation, and use .has() to check for key existence before accessing.

Iterate over maps with for k, v in map to get both keys and values. The .keys and .values properties give you lists of all keys or values.

main()
    // Create a map
    scores! = {"alice": 95, "bob": 87, "carol": 92}

    // Access by key
    print(scores["alice"])   // 95
    print(scores.len)        // 3

    // Check for key
    if scores.has("bob")
        print("Bob's score: {scores["bob"]}")

    // Add / update
    scores["dave"] = 88

    // Remove
    scores.remove!("bob")

    // Get all keys and values
    print(scores.keys)       // ["alice", "carol", "dave"]
    print(scores.values)     // [95, 92, 88]

    // Iterate over key-value pairs
    for name, score in scores
        print("{name}: {score}")mix

08Sets

Sets are unordered collections of unique values. Create them with the set{} syntax. Sets support the classic mathematical operations: union, intersection, and difference.

Like lists, mutation methods use the ! suffix. Use .has() for membership testing and .values to get the elements as a list.

main()
    // Create a set
    fruits! = set{"apple", "banana", "cherry"}
    empty = set{}

    // Add and remove
    fruits.add!("date")
    fruits.remove!("banana")

    // Check membership
    print(fruits.has("apple"))    // true
    print(fruits.has("banana"))   // false
    print(fruits.len)             // 3

    // Get all values as a list
    print(fruits.values)

    // Set operations
    a = set{1, 2, 3, 4}
    b = set{3, 4, 5, 6}

    both = a.union(b)            // {1, 2, 3, 4, 5, 6}
    common = a.intersect(b)       // {3, 4}
    only_a = a.diff(b)            // {1, 2}

    // Iterate over a set
    for item in fruits
        print("fruit: {item}")mix

09Shapes

Shapes are MIX's structured data types, similar to structs or classes. They have fields, methods, and computed properties. Define a shape with the shape keyword, then list fields and methods indented beneath it.

Construct a shape using named arguments Shape(field: value) or positional arguments Shape(val1, val2). Methods have implicit access to self fields -- just use the field name directly. Computed fields are zero-parameter methods that can be called without parentheses.

shape Player
    name: str
    health: int
    max_health: int

    // Computed field (no parens needed to call)
    is_alive() -> bool
        health > 0

    // Method with parameter
    take_damage(amount: int) -> Player
        new_hp = health - amount
        if new_hp < 0
            new_hp = 0
        Player(name: name, health: new_hp, max_health: max_health)

    // Computed field for display
    status() -> str
        "{name}: {health}/{max_health} HP"

main()
    // Named construction
    p = Player(name: "Hero", health: 100, max_health: 100)

    // Field access
    print(p.name)           // "Hero"
    print(p.health)         // 100

    // Computed field (no parens)
    print(p.is_alive)       // true
    print(p.status)         // "Hero: 100/100 HP"

    // Methods return new shapes (immutable by default)
    p2 = p.take_damage(35)
    print(p2.status)        // "Hero: 65/100 HP"

    // Positional construction
    p3 = Player("Mage", 80, 80)
    print(p3.status)        // "Mage: 80/80 HP"mix

10Operator Overloading

Shapes can define custom behavior for arithmetic and comparison operators. Define a method named after the operator (+, -, *, /, %, ==, !=) that takes one parameter (the right-hand operand) and returns the result type.

This makes mathematical types like vectors, matrices, and complex numbers feel natural to use with standard operator syntax.

shape Vec2
    x, y: float

    +(other: Vec2) -> Vec2
        Vec2(x: x + other.x, y: y + other.y)

    -(other: Vec2) -> Vec2
        Vec2(x: x - other.x, y: y - other.y)

    *(scalar: float) -> Vec2
        Vec2(x: x * scalar, y: y * scalar)

    ==(other: Vec2) -> bool
        x == other.x and y == other.y

    length_sq() -> float
        x * x + y * y

    to_string() -> str
        "({x}, {y})"

main()
    a = Vec2(x: 3.0, y: 4.0)
    b = Vec2(x: 1.0, y: 2.0)

    c = a + b
    print("a + b = {c}")       // (4.0, 6.0)

    d = a - b
    print("a - b = {d}")       // (2.0, 2.0)

    e = a * 2.0
    print("a * 2 = {e}")       // (6.0, 8.0)

    print("a == b: {a == b}") // false
    print("length^2 = {a.length_sq}")mix

11Tagged Unions

Tagged unions let a shape hold one of several named variants, each with different data. Define variants inside a shape as indented alternatives. This is MIX's approach to algebraic data types / sum types.

Construct variants using ShapeName.Variant(data) and destructure them with match to extract the inner values safely.

shape Expr
    Num(value: int)
    Add(left: Expr, right: Expr)
    Neg(inner: Expr)

eval(e: Expr) -> int
    match e
        Num(value) => value
        Add(left, right) => eval(left) + eval(right)
        Neg(inner) => -eval(inner)

main()
    // Build: -(3 + 4)
    expr = Expr.Neg(Expr.Add(
        Expr.Num(3),
        Expr.Num(4)
    ))

    result = eval(expr)
    print("result = {result}")  // result = -7mix

12Match Expressions

In addition to being used as a statement (for branching), match can be used as an expression that produces a value. When used as an expression, each arm evaluates to a value and the result of the matching arm is returned.

This is useful for assigning the result of pattern matching to a variable or returning it from a function. The wildcard _ catches any unmatched value.

describe(n: int) -> str
    match n
        0 => "zero"
        1 => "one"
        2 => "two"
        _ => "many"

classify(score: int) -> str
    match score
        90 => "A"
        80 => "B"
        70 => "C"
        _ => "F"

main()
    // Match as expression (assign result to variable)
    x = 42
    label = match x
        0 => "nothing"
        42 => "the answer"
        _ => "something else"

    print(label)  // "the answer"

    // Match as last expression in function
    print(describe(1))    // "one"
    print(describe(99))   // "many"
    print(classify(90))   // "A"mix

13Error Handling

MIX uses result types for error handling instead of exceptions. A function that can fail uses fail "message" to signal an error and done value to return success. Functions with ~ and fail are automatically wrapped in a result type.

Callers handle errors with else for a fallback value, or ? to propagate the error upward. This makes the error path explicit and visible at every call site.

// A function that can fail
divide(a: float, b: float) -> float ~
    if b == 0.0
        fail "division by zero"
    done a / b

// Parse an integer from a string
parse_positive(s: str) -> int ~
    n = parse_int(s)?          // propagate parse error with ?
    if n <= 0
        fail "must be positive"
    done n

main()
    // Use else for fallback value
    result = divide(10.0, 3.0) else 0.0
    print("10 / 3 = {result}")

    // Fallback on error
    bad = divide(10.0, 0.0) else -1.0
    print("10 / 0 = {bad}")     // 10 / 0 = -1.0

    // Parse with fallback
    age = parse_positive("25") else 0
    print("age = {age}")         // age = 25mix

14Optionals

Optional types represent values that may or may not exist. Declare an optional with T? (e.g., int?, str?). The special value none represents the absence of a value.

Unwrap an optional with else to provide a default, or use match to handle both cases. This prevents null pointer errors at compile time.

// A function that might not find a result
find_index(items: [str], target: str) -> int?
    for i in 0..items.len
        if items[i] == target
            done i
    none

main()
    names = ["alice", "bob", "carol"]

    // Unwrap with else fallback
    idx = find_index(names, "bob") else -1
    print("bob at index: {idx}")    // bob at index: 1

    // Not found -- falls back to default
    idx2 = find_index(names, "dave") else -1
    print("dave at index: {idx2}")   // dave at index: -1

    // Use in conditional
    result = find_index(names, "carol")
    if result != none
        print("found carol")mix

15Generics

Generics let you write functions and shapes that work with any type. Use @T to declare a type parameter. You can have multiple type parameters and constrain them with has to require specific capabilities.

Generic shapes work the same way -- the type parameter is resolved when you construct an instance.

// Generic identity function
identity(@T)(value: T) -> T
    value

// Generic pair
shape Pair(@T)
    first: T
    second: T

    swap() -> Pair(T)
        Pair(first: second, second: first)

// Generic stack with constraint
shape Stack(@T)
    items: [T]

    push!(item: T)
        items.push!(item)

    pop!() -> T
        items.pop!()

    is_empty() -> bool
        items.len == 0

main()
    // Generic function
    x = identity(42)
    s = identity("hello")

    // Generic shape
    p = Pair(first: 1, second: 2)
    p2 = p.swap()
    print("{p2.first}, {p2.second}")  // 2, 1

    // Generic stack
    stack! = Stack(items: [1, 2, 3])
    stack.push!(4)
    top = stack.pop!()
    print("top = {top}")  // top = 4mix

16Modules

In MIX, each file is a module. Use pub to export functions and shapes from a module, and use to import them in another file. The module name is derived from the filename.

Only items marked pub are visible outside the module. Everything else is private by default, giving you clean encapsulation without boilerplate.

math_utils.mix

// Export with pub
pub clamp(val: int, lo: int, hi: int) -> int
    if val < lo
        done lo
    if val > hi
        done hi
    val

pub lerp(a: float, b: float, t: float) -> float
    a + (b - a) * t

pub shape Rect
    x, y, w, h: float

    area() -> float
        w * hmath_utils.mix

main.mix

use math_utils

main()
    // Use imported functions
    val = math_utils.clamp(150, 0, 100)
    print("clamped: {val}")    // clamped: 100

    t = math_utils.lerp(0.0, 10.0, 0.5)
    print("lerp: {t}")         // lerp: 5.0

    // Use imported shape
    r = math_utils.Rect(x: 0.0, y: 0.0, w: 10.0, h: 5.0)
    print("area = {r.area}")   // area = 50.0main.mix

17Standard Library

MIX ships with a small standard library organized into modules. Import them with use std.module_name. The standard library focuses on common operations that every program needs.

Available modules include std.math for mathematical constants and functions, std.string for advanced string operations, and std.collections for data structures like stacks and queues.

use std.math
use std.string
use std.collections

main()
    // std.math -- constants and functions
    print(math.PI)                  // 3.14159...
    print(math.hypot(3.0, 4.0))    // 5.0
    print(math.sign(-42.0))        // -1.0

    // std.string -- extra string utilities
    print(string.capitalize("hello"))  // "Hello"
    print(string.reverse("abc"))      // "cba"
    print(string.pad_left("42", 5, "0")) // "00042"

    // std.collections -- Stack
    s! = collections.Stack(items: [1, 2, 3])
    s.push!(4)
    top = s.pop!()
    print("top = {top}")             // top = 4

    // std.collections -- Queue
    q! = collections.Queue(items: ["a", "b"])
    q.push!("c")
    front = q.pop!()
    print("front = {front}")         // front = "a"mix

18Concurrency

MIX provides lightweight concurrency with go for spawning tasks and wait for collecting results. Shared state is managed with the shared keyword, which wraps a value in a thread-safe container.

Read shared values with .read() and mutate them with .update!(). The runtime handles locking automatically. This design prevents data races by making shared access explicit.

compute(n: int) -> int
    n * n

main()
    // Shared mutable counter
    counter = shared int(0)

    // Spawn concurrent tasks
    t1 = go compute(10)
    t2 = go compute(20)
    t3 = go compute(30)

    // Wait for results
    r1 = wait t1
    r2 = wait t2
    r3 = wait t3

    print("results: {r1}, {r2}, {r3}")
    // results: 100, 400, 900

    // Update shared state
    counter.update!(r1 + r2 + r3)
    print("total = {counter.read()}")
    // total = 1400mix

19Memory Management

MIX provides three tools for precise memory control: defer for automatic cleanup, unsafe for raw pointer operations, and zone for scoped arena allocation.

The defer keyword schedules a function to run when the current scope exits, regardless of how it exits. This is ideal for closing files and releasing resources. The unsafe block disables safety checks for low-level pointer work. The zone block creates a memory arena that is freed all at once when the block ends.

main()
    // defer -- cleanup runs at scope exit
    handle = file_open("data.txt")
    defer file_close(handle)
    // ... use handle ... file_close runs automatically

    // Multiple defers run in reverse order (LIFO)
    defer print("first defer")
    defer print("second defer")
    // prints "second defer" then "first defer"

    // unsafe -- raw pointer operations
    unsafe
        ptr = malloc(64)
        // ... work with raw memory ...
        free(ptr)

    // zone -- scoped arena allocation
    zone
        // all allocations in this block use arena
        data = [1, 2, 3, 4, 5]
        print("zone data: {data}")
    // arena freed here -- all zone allocations gonemix

20C Interop

MIX provides transparent C interop via use c — import a C header directly and all its functions, constants, and enums become available as native MIX symbols. No manual declarations, no binding generator step. Just use c and start calling C.

Transparent Import (use c)

// Import a C header — everything is available immediately
use c "SDL3/SDL.h" link "SDL3"

main()
    SDL_Init(SDL_INIT_VIDEO)     // enum constant, auto-imported
    defer SDL_Quit()

    win = SDL_CreateWindow("MIX", 800, 600, 0)
    print("Window created!")
    SDL_DestroyWindow(win)mix

The link "SDL3" clause automatically adds -lSDL3 to the linker. Compile with just:

mix build game.mixshell

What gets imported automatically:

  • Function prototypes (all C functions become callable)
  • #define constants (integers and floats, hex preserved)
  • C enums (enum Name {} and typedef enum {} Name)
  • typedef struct shapes (field names and types)

Memory Builtins

When working with C libraries, you often need raw memory. MIX provides built-in functions:

main()
    // Allocate a buffer for C struct data (zeroed)
    event = bytes(128)

    // Read a uint32 from the buffer (e.g., SDL_Event.type)
    event_type = peek_u32(event)

    // General-purpose allocation
    buf = alloc(1024)

    // Free when done
    free_mem(event)
    free_mem(buf)mix

Manual Declarations (extern)

For fine-grained control, you can declare C functions manually:

// Declare external C functions
extern "libc"
    puts(s: str) -> int
    getenv(name: str) -> str
    abs(n: int) -> int

main()
    puts("Hello from C!")
    home = getenv("HOME")
    print("home = {home}")mix

Offline Binding Generation (--bind)

For large libraries, you can pre-generate bindings to a .mix file:

# Generate MIX bindings from a C header
mix --bind SDL3/SDL.h -o sdl3.mix --lib SDL3shell

21Lambdas

Lambdas are anonymous functions defined inline with the => arrow syntax. Single-parameter lambdas use x => expr, and multi-parameter lambdas use (a, b) => expr. Lambdas can be passed as arguments to higher-order functions.

This makes functional patterns like map, filter, and reduce concise and readable.

// Higher-order function that takes a function argument
apply_twice(x: int, f: (int) -> int) -> int
    f(f(x))

transform(items: [int], f: (int) -> int) -> [int]
    [f(x) for x in items]

main()
    // Single-parameter lambda
    doubled = apply_twice(3, x => x * 2)
    print("doubled: {doubled}")  // doubled: 12

    // Multi-parameter lambda
    add = (a, b) => a + b
    print("sum: {add(3, 4)}")    // sum: 7

    // Lambda with higher-order function
    nums = [1, 2, 3, 4, 5]
    squares = transform(nums, x => x * x)
    print(squares)   // [1, 4, 9, 16, 25]

    // Inline filter with comprehension
    big = [x for x in nums if x > 3]
    print(big)       // [4, 5]mix

22Compile-Time Features

MIX supports compile-time constants and platform detection using the @ prefix. Use @const to define values evaluated at compile time. The @os and @arch directives let you write platform-specific code that is resolved during compilation.

Dead branches (where the compile-time condition is false) are eliminated from the final binary entirely, with no runtime cost.

// Compile-time constant
@const MAX_PLAYERS = 64
@const VERSION = "1.0.0"

main()
    print("version {VERSION}")
    print("max players: {MAX_PLAYERS}")

    // Platform detection
    if @os == "macos"
        print("Running on macOS")
    else if @os == "linux"
        print("Running on Linux")

    // Architecture detection
    if @arch == "aarch64"
        print("ARM 64-bit")
    else if @arch == "x86_64"
        print("x86 64-bit")mix

23File I/O

MIX provides built-in functions for reading and writing files. Use file_read_all and file_write_all for simple one-shot operations. For more control, use file_open, file_write, and file_close with defer for automatic cleanup.

Check for file existence with file_exists before reading. These functions work with the result type, so use else or ? for error handling.

main()
    // Write a file (one-shot)
    file_write_all("hello.txt", "Hello, world!\n")

    // Check if file exists
    if file_exists("hello.txt")
        print("file exists")

    // Read entire file
    content = file_read_all("hello.txt") else ""
    print("content: {content}")

    // File handle with defer for cleanup
    f = file_open("log.txt")
    defer file_close(f)

24OS and System

MIX includes built-in functions for interacting with the operating system. Run shell commands, read environment variables, work with the filesystem, and handle command-line arguments -- all without importing any modules.

These functions are available globally. Use shell() to run commands and get the exit code, or shell_output() to capture stdout as a string.

main()
    // Run a shell command (returns exit code)
    code = shell("echo 'hello from shell'")
    print("exit code: {code}")

    // Capture command output
    output = shell_output("uname -m")
    print("arch: {output}")

    // Environment variables
    home = env("HOME")
    print("home: {home}")

    // Current working directory
    cwd = getcwd()
    print("cwd: {cwd}")

    // Create directory (recursive, like mkdir -p)
    mkdir("output/logs")

    // List directory contents
    files = list_dir(".")
    for f in files
        print("  {f}")

    // Check file existence
    if file_exists("build.mix")
        print("build.mix found")

    // Command-line arguments
    argv = args()
    for arg in argv
        print("arg: {arg}")

    // Exit with code
    exit(0)mix

25Build System

MIX has a built-in build system based on the Project shape. Create a build.mix file in your project root that defines a Project with your build configuration. Running mix build (with no file argument) automatically finds and executes build.mix.

The Project shape supports fields for the entry file, output binary name, C libraries to link, library search paths, include paths, compiler flags, and a debug mode toggle. The .build() method compiles the project using these settings.

// build.mix -- project build configuration

main()
    project = Project(
        name: "my_game",
        entry: "src/game.mix",
        output: "game",
        libs: ["SDL3", "m"],
        lib_paths: ["/usr/local/lib"],
        include_paths: ["/usr/local/include"],
        flags: ["-O2"],
        debug: false
    )

    project.build()build.mix

Then build and run your project from the command line:

# Run build.mix to compile the project
mix build

# Or compile and run in one step
mix run

# The output binary is placed in the current directory
./gameshell

Default values for missing fields: string fields default to "", list fields default to [], and debug defaults to false. You only need to specify the fields relevant to your project.

// Minimal build.mix -- just entry and output

main()
    project = Project(
        name: "hello",
        entry: "hello.mix",
        output: "hello"
    )

    project.build()build.mix
