Language Reference

Complete overview of the mix programming language -- types, syntax, standard library, and tooling.

01Type System

mix is statically typed. Every value has a known type at compile time. Types are inferred where possible, but can always be annotated explicitly.

Optionals

Append ? to any type to make it optional. Use none for absent values and else for fallback.

name: str? = none

// unwrap with else
display = name else "anonymous"

// check before use
if name != none
    print("Hello, {name}")mix

Type Checking

The compiler checks types at function call sites, catching mismatches at compile time instead of failing at runtime.

Argument type checking

add(a: int, b: int) -> int
    a + b

main()
    add("x", "y")   // error: argument 1 of 'add': expected int, got strmix

Collection element types

process(nums: [int]) ~
    print(nums)

main()
    process(["a", "b"])   // error: argument 1 of 'process': expected [int], got [str]mix

Shape type checking

shape Vec2
    x, y: float

shape Color
    r, g, b: int

normalize(v: Vec2) -> Vec2
    // ...

main()
    c = Color(r: 255, g: 0, b: 0)
    normalize(c)   // error: argument 1 of 'normalize': expected Vec2, got Colormix

Argument count checking

add(1)         // error: 'add' expects 2 argument(s), got 1
add(1, 2, 3)   // error: 'add' expects 2 argument(s), got 3mix

Implicit numeric promotion

Integers are automatically promoted to floats when passed to float parameters. No explicit cast needed.

scale(v: Vec2, factor: float) -> Vec2
    // ...

main()
    v = Vec2(x: 3.0, y: 4.0)
    scale(v, 2)   // OK: int 2 promoted to float 2.0mix

02Variables

Immutable by Default

Variables are immutable unless marked with !.

x = 42           // immutable -- cannot be reassigned
y! = 10          // mutable -- can be reassigned
y! = 20          // OKmix

Every mutation requires the ! marker at the point of change, not just at declaration.

Typed Declarations

x: float = 3.14
name: str = "mix"
flag: bool = truemix

Compound Assignment

count! = 0
count! += 5      // count is now 5
count! -= 2      // count is now 3
count! *= 4      // count is now 12
count! /= 3      // count is now 4mix

03Functions

Basic Declaration

add(a: int, b: int) -> int
    a + b    // implicit return -- last expression is the return valuemix

No Return Value

greet(name: str) ~
    print("Hello, {name}!")mix

Early Return with done

first_positive(nums: [int]) -> int
    for n in nums
        if n > 0
            done n
    0mix

Side Effects Marker ~

Functions that perform I/O, use fail, or have other side effects must be marked with ~. The ~ goes after the return type or after the closing paren if there is no return type.

save(data: str, path: str) -> bool ~
    file_write_all(path, data)
    done truemix

Mutation Marker !

Functions that mutate their arguments use the ! marker.

double_all!(nums: [int])
    for i in 0..nums.len
        nums[i]! = nums[i] * 2mix

Recursion

factorial(n: int) -> int
    if n <= 1
        done 1
    n * factorial(n - 1)mix

04Control Flow

if / else if / else

if x > 100
    print("big")
else if x > 10
    print("medium")
else
    print("small")mix

while

i! = 0
while i < 10
    print("{i}")
    i! += 1mix

for Loops

// exclusive range: 0, 1, 2, 3, 4
for i in 0..5
    print("{i}")

// inclusive range: 0, 1, 2, 3, 4, 5
for i in 0..=5
    print("{i}")

// iterate over a list
for item in items
    print("{item}")

// iterate over a map
for key, value in config
    print("{key} = {value}")

// iterate over a set
for item in names
    print("{item}")mix

break and continue

for i in 0..100
    if i % 2 == 0
        continue    // skip even numbers
    if i > 20
        break       // stop at 20
    print("{i}")mix

match

Pattern matching with exhaustive checking.

match command
    "start" => start_server()
    "stop"  => stop_server()
    "help"  => show_help()
    _       => print("Unknown command: {command}")mix

Match as Expression

Match arms can produce values, making match usable as an expression.

describe(n: int) -> str
    match n
        0  => "zero"
        1  => "one"
        _  => "many"mix

Match with Destructuring

match event
    Click(x, y)     => handle_click(x, y)
    KeyPress(key)   => handle_key(key)
    Resize(w, h)    => handle_resize(w, h)
    _               => print("unhandled event")mix

05Shapes

Shapes are mix's structured data types -- similar to structs, but with built-in methods, computed fields, operator overloading, and tagged unions.

Declaration

shape Vec2
    x: float
    y: floatmix

Construction

// named fields
pos = Vec2(x: 3.0, y: 4.0)

// positional
pos = Vec2(3.0, 4.0)mix

Methods

Methods have implicit self. Fields are accessed directly by name.

shape Vec2
    x: float
    y: float

    length() -> float
        sqrt(x * x + y * y)

    scale!(factor: float)
        x! = x * factor
        y! = y * factormix

Computed Fields

No-parens methods that look like field access.

shape Circle
    radius: float

    area -> float
        3.14159 * radius * radius

c = Circle(radius: 5.0)
print("{c.area}")   // accessed like a fieldmix

Operator Overloading

shape Vec2
    x: float
    y: float

    +(other: Vec2) -> Vec2
        Vec2(x + other.x, y + other.y)

    ==(other: Vec2) -> bool
        x == other.x && y == other.y

a = Vec2(1.0, 2.0)
b = Vec2(3.0, 4.0)
c = a + b   // Vec2(4.0, 6.0)mix

Overloadable operators: + - * / % == != < > <= >=

Tagged Unions

shape Expr
    Num(float)
    Add(Expr, Expr)
    Mul(Expr, Expr)

eval(e: Expr) -> float
    match e
        Num(n)    => n
        Add(a, b) => eval(a) + eval(b)
        Mul(a, b) => eval(a) * eval(b)mix

Public Shapes

    port: intmix

07Error Handling

mix uses a Result-based error model. Functions that can fail are automatically wrapped in a Result type.

Signaling Failure

parse_port(s: str) -> int ~
    if s.len == 0
        fail "empty string"
    // ... parsing logic ...
    done portmix

Result Auto-Wrapping

When a function has ~, a non-void return type, and its body contains fail or ?, the return type is automatically wrapped in Result. Callers receive either a success value or an error.

Handling Errors with else

// provide a fallback value on failure
port = parse_port("8080") else 3000

// always succeeds -- port is 8080 or 3000
print("Using port {port}")mix

Error Propagation with ?

The ? operator propagates errors to the caller automatically.

load_config(path: str) -> Config ~
    content = file_read_all(path)?
    config = parse_config(content)?
    done configmix

Early Success with done

find_user(id: int) -> User ~
    for user in users
        if user.id == id
            done user
    fail "user not found: {id}"mix

10Concurrency

Shared State

The shared keyword creates a thread-safe value that can be read and updated atomically.

counter! = shared int(0)

// read the current value
current = counter.read()

// update atomically
counter.update!(v => v + 1)mix

Spawning Tasks

compute(x: int) -> int
    x * x

task = go compute(42)
result = wait task   // blocks until done, returns 1764mix

Parallel Work

t1 = go fetch_users()
t2 = go fetch_posts()
t3 = go fetch_comments()

users = wait t1
posts = wait t2
comments = wait t3mix

11Memory & Safety

Unsafe Blocks

Low-level operations like pointer dereferencing and arithmetic require an unsafe block.

unsafe
    ptr: *int = raw_alloc(8)
    *ptr = 42
    val = *ptr
    ptr = ptr + 1   // pointer arithmeticmix

Zones

Zone-based memory management for grouping allocations.

zone
    // all allocations in this block are grouped
    data = process(input)
    result = transform(data)
// memory freed here

// named zones
zone:render
    frame = build_frame()
    draw(frame)mix

Defer

Deferred statements run in LIFO order when the enclosing scope exits.

process_file(path: str) ~
    handle = file_open(path)
    defer file_close(handle)

    // work with file...
    // file_close is called automatically on exitmix

Project Shape

Define build configuration using the built-in Project shape in a build.mix file.

    project = Project(
        name:          "my_app",
        entry:         "src/main.mix",
        output:        "build/my_app",
        libs:          ["SDL2", "SDL2_image"],
        lib_paths:     ["/opt/homebrew/lib"],
        include_paths: ["/opt/homebrew/include"],
        flags:         ["-O2"],
        debug:         false
    )

    project.build()mix

Build Script Example

Build scripts are regular mix programs. You can use any language feature -- conditionals, loops, OS builtins -- to create custom build logic.

main() ~
    // clean build directory
    if file_exists("build")
        shell("rm -rf build")
    mkdir("build")

    // detect platform-specific libs
    libs! = ["SDL2"]
    @os == "linux"
        libs.push!("GL")
    @os == "macos"
        libs.push!("OpenGL")

    project = Project(
        name:   "game",
        entry:  "src/main.mix",
        output: "build/game",
        libs:   libs,
        debug:  true
    )

    project.build()
    print("Build complete!")mix
