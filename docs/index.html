03Shapes, Methods & Generics

shape Circle
    radius: float

    area() -> float
        3.14159 * radius * radius

    circumference() -> float
        2.0 * 3.14159 * radius

c = Circle(radius: 5.0)
print(c.area)             // 78.5397 (computed field)
print(c.circumference())  // 31.4159mix

Methods access fields directly by name. No self. prefix needed. Zero-param methods can be called without ().

05Concurrency

MIX provides minimal concurrency primitives built on pthreads. No -lpthread needed on macOS.

shared — Thread-Safe Values

counter! = shared int(0)

counter.update!(v => v + 1)
counter.update!(v => v + 1)
counter.update!(v => v + 1)

print(counter.read())    // 3mix

shared int(val) wraps a value with a mutex. .read() and .update!(fn) lock automatically.

go / wait — Task Spawning

compute(x: int) -> int
    x * x + 1

main()
    t1 = go compute(5)
    t2 = go compute(10)
    t3 = go compute(3)

    print(wait t1)        // 26
    print(wait t2)        // 101
    print(wait t3)        // 10mix

go fn(args) spawns a new pthread. wait handle joins and returns the result. Functions with 0-8 args supported.

09Tooling

C Header Bindings (--bind)

Auto-generate MIX extern blocks from C headers. Handles SDL3 (1,400+ functions), Raylib (580+ functions), and any C library. Shows progress for directory mode.

# Single header — completes in ~0.1s even for SDL3
./build/mix --bind /opt/homebrew/include/SDL3/SDL.h -o sdl3.mix --lib SDL3

# Entire directory with progress output
./build/mix --bind /opt/homebrew/include/SDL3/ -o sdl3.mix --lib SDL3
# mix: [1/86] SDL_tray.h
# mix: [2/86] SDL_misc.h
# ...
# mix: wrote 2129 functions, 4096 constants

# Check version
./build/mix --version  # mix 0.1.0 (2026-03-04)

Given a C header like:

int SDL_Init(uint32_t flags);
void SDL_Quit(void);
char *SDL_GetError(void);
#define SDL_INIT_VIDEO 0x00000020c

The binding generator produces:

extern "SDL3"
    SDL_Init(flags: uint32) -> int32 ~
    SDL_Quit() ~
    SDL_GetError() -> *byte ~

@const SDL_INIT_VIDEO = 32mix

Type Mapping

int→int32, double→float, uint8_t→uint8, all pointers→*byte, void→no return

Preprocessor

Runs cc -E to expand macros and includes. Extracts #define integer/float constants.

Directory Mode

Process an entire folder of .h files. Deduplicates functions across headers.

Debug Info (--debug)

Compile with DWARF line numbers for source-level debugging with lldb or gdb:

./build/mix program.mix -o program --debug

Emits QBE dbgfile/dbgloc directives → .file/.loc assembly directives → DWARF line table in the binary.

MIX ships with a Language Server Protocol (LSP) implementation that gives you a rich editing experience in any LSP-capable editor.

Context-aware completions for symbols, keywords, shape fields, methods, and even inside string interpolation.

Building the LSP

The LSP server mix-lsp is built automatically alongside the compiler:

make              # builds both build/mix and build/mix-lsp

Architecture

The LSP server is written in C, reusing the compiler's lexer, parser, and semantic analyzer. Each open document gets its own arena, and the full lex→parse→sema pipeline re-runs on every keystroke (fast enough for typical MIX files). Communication uses JSON-RPC over stdin/stdout with a hand-rolled JSON parser — zero external dependencies.

10Tutorial: Build a Number Guessing Game

Let's build a complete program step by step — a number guessing game that demonstrates variables, functions, control flow, shapes, and string interpolation.

Step 1: Hello World

Every MIX program starts with main(). The body is indented 4 spaces. No semicolons, no braces.

main()
    print("Welcome to the Number Game!")mix
./build/mix game.mix -o game && ./game
# Welcome to the Number Game!

Step 2: Functions

Functions are defined by name + params. The last expression is the return value — no return keyword.

square(x: int) -> int
    x * x

clamp(val: int, lo: int, hi: int) -> int
    if val < lo
        done lo          // early exit
    if val > hi
        done hi
    val                  // last expression = return value

main()
    print(square(7))      // 49
    print(clamp(15, 0, 10)) // 10mix

Step 3: Mutable Variables & Loops

Variables are immutable by default. Add ! to make them mutable.

main()
    total! = 0              // mutable
    for i in 1..11          // 1 to 10
        total! += i
    print("Sum 1-10 = {total}")  // Sum 1-10 = 55mix

Step 4: Shapes (Structs)

Shapes are MIX's structs. Methods access fields directly — no self. prefix.

shape Player
    name: str
    score: int

    greet() ~
        print("Player: {name}, Score: {score}")

main()
    p = Player(name: "Alice", score: 0)
    p.greet()  // Player: Alice, Score: 0mix

Step 5: Lists & Comprehensions

Lists use [...] syntax. Comprehensions make new lists from existing ones.

main()
    nums = [1, 2, 3, 4, 5]
    doubled = [x * 2 for x in nums]
    print(doubled)           // [2, 4, 6, 8, 10]

    big = [x for x in nums if x > 3]
    print(big)               // [4, 5]mix

Step 6: Match & Optionals

Pattern matching with match. Optionals with T? and else for safe unwrapping.

describe(n: int) -> str
    match n
        1 => "one"
        2 => "two"
        3 => "three"
        _ => "many"

find_first_even(items: [int]) -> int?
    for item in items
        if item % 2 == 0
            done item
    none

main()
    print(describe(2))              // two
    nums = [1, 3, 5, 4, 7]
    result = find_first_even(nums) else -1
    print(result)                   // 4mix

Step 7: Concurrency

Spawn work on threads with go, collect results with wait.

heavy_compute(n: int) -> int
    total! = 0
    for i in 0..n
        total! += i * i
    total

main()
    // Run three computations in parallel
    t1 = go heavy_compute(1000)
    t2 = go heavy_compute(2000)
    t3 = go heavy_compute(3000)

    r1 = wait t1
    r2 = wait t2
    r3 = wait t3
    print("Results: {r1}, {r2}, {r3}")mix

Step 8: C Interop

Call any C function directly. Generate bindings automatically with --bind.

// Step 1: Generate bindings from C headers
// $ ./build/mix --bind /opt/homebrew/include/SDL3/SDL.h -o sdl3.mix --lib SDL3

// Step 2: Use the generated bindings in your program
extern "SDL3"
    SDL_Init(flags: uint32) -> int32 ~
    SDL_Quit() ~

@const SDL_INIT_VIDEO = 32

main()
    SDL_Init(SDL_INIT_VIDEO)
    print("SDL initialized!")
    defer SDL_Quit()      // cleanup runs automaticallymix

Compile with: ./build/mix game.mix -o game -lSDL3

11SDL3 Example

A native SDL3 window — cornflower blue — in 40 lines of MIX:

extern "SDL3"
    SDL_Init(flags: uint32) -> int32 ~
    SDL_Quit() ~
    SDL_CreateWindow(title: *byte, w: int32, h: int32, flags: uint64) -> *byte ~
    SDL_DestroyWindow(win: *byte) ~
    SDL_CreateRenderer(win: *byte, name: *byte) -> *byte ~
    SDL_SetRenderDrawColor(rnd: *byte, r: uint8, g: uint8, b: uint8, a: uint8) -> int32 ~
    SDL_RenderClear(rnd: *byte) -> int32 ~
    SDL_RenderPresent(rnd: *byte) -> int32 ~
    SDL_PollEvent(event: *byte) -> int32 ~
    SDL_Delay(ms: uint32) ~

@const SDL_INIT_VIDEO = 0x00000020
@const SDL_EVENT_QUIT = 0x100

main()
    SDL_Init(SDL_INIT_VIDEO)
    win = SDL_CreateWindow("MIX + SDL3", 800, 600, 0)
    rnd = SDL_CreateRenderer(win, 0)
    event = mix_bytes(128)
    running! = 1

    while running > 0
        while SDL_PollEvent(event) > 0
            etype = mix_peek_u32(event)
            if etype == SDL_EVENT_QUIT
                running! = 0

        SDL_SetRenderDrawColor(rnd, 100, 149, 237, 255)
        SDL_RenderClear(rnd)
        SDL_RenderPresent(rnd)
        SDL_Delay(16)mix
